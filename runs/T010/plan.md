## Objective

Bootstrap a React + Vite frontend inside the repository under a dedicated `frontend/` directory so the project can be started locally and render a minimal application shell. This creates the foundation for later tickets (CRA calendar, API integration, PDF download) without altering any existing agent/factory files.

## Included

- Create a new top-level directory `frontend/` that isolates the React application from the AI Dev Factory folders (`ai/`, `docs/`, `prompts/`, `runs/`, `tickets/`).
- Initialize a Vite React project inside `frontend/` using the `react` template (JavaScript, not TypeScript, to keep scope minimal). The following files are created by the Vite template and must be committed:
  - `frontend/package.json` declaring dependencies (`react`, `react-dom`) and devDependencies (`vite`, `@vitejs/plugin-react`, `@types/react`, `@types/react-dom` if the template adds them, `eslint` only if the template adds it by default). Do not add extra libraries.
  - `frontend/package.json` scripts: `dev`, `build`, `preview` (as generated by the Vite template).
  - `frontend/vite.config.js` with the default `@vitejs/plugin-react` configuration.
  - `frontend/index.html` as the Vite entry HTML.
  - `frontend/src/main.jsx` mounting `<App />` on `#root`.
  - `frontend/src/App.jsx` — replace the default template content with a minimal application shell: a single component rendering a `<header>` with the app title (e.g. "Timizer Like") and a `<main>` with a short placeholder text ("Bootstrap OK"). No routing, no state, no API calls.
  - `frontend/src/App.css` and `frontend/src/index.css` kept minimal (default styles from template acceptable, or trimmed to a plain reset). No visual polish.
  - `frontend/public/` folder from the template (may keep the default `vite.svg` or leave empty).
- Add `frontend/.gitignore` covering `node_modules/`, `dist/`, and local Vite/log files.
- Add a short `frontend/README.md` describing the two commands `npm install` and `npm run dev`, and the resulting local URL (default `http://localhost:5173`).
- Verify (via a local `npm install` + `npm run dev`) that the shell loads, then commit the source files. Do NOT commit `node_modules/` or `dist/`.
- Confirm that no file outside `frontend/` is modified, except optionally the root `.gitignore` if it needs a `frontend/node_modules/` entry (only if a root `.gitignore` already exists; otherwise skip).

## Excluded

- CRA calendar UI and any month/day input logic.
- API client, HTTP calls, or backend integration.
- PDF download flow.
- Styling framework (Tailwind, MUI, etc.) and any polish beyond the Vite defaults.
- Authentication, routing (React Router), state management (Redux, Zustand), and testing setup (Vitest, Jest, RTL).
- TypeScript migration.
- Backend or server code.
- CI configuration for the frontend (linting workflow, deploy, etc.).
- Docker / containerization.

## Acceptance criteria

- The directory `frontend/` exists at the repository root and contains `package.json`, `vite.config.js`, `index.html`, `src/main.jsx`, `src/App.jsx`, and a `.gitignore`.
- `frontend/package.json` declares `react`, `react-dom` as dependencies and `vite`, `@vitejs/plugin-react` as devDependencies, and defines the scripts `dev`, `build`, `preview`.
- Running `cd frontend && npm install && npm run dev` starts the Vite dev server without errors and serves an HTTP page at the default local URL.
- Opening that page shows the minimal shell: the app title "Timizer Like" and a placeholder text (e.g. "Bootstrap OK"). No console errors in the browser.
- Running `cd frontend && npm run build` completes successfully and produces a `dist/` directory.
- `node_modules/` and `dist/` are not committed (ignored by `frontend/.gitignore`).
- No file outside the `frontend/` directory is modified, except at most an added ignore entry in the root `.gitignore`. In particular, `ai/`, `docs/`, `prompts/`, `runs/`, `tickets/` remain unchanged.
- The repository still contains all previously existing files and directories intact.
